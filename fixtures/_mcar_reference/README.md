# Little's MCAR reference suite

Three small synthetic datasets (`ozone.csv`, `iris_mar.csv`, `nhanes.csv`)
plus an `expected.json` sidecar pinning the d² statistic, degrees of
freedom, and p-value of Little's MCAR test for each. Used by the
regression test `test_mcar_matches_reference_suite` in
`tests/test_missingness.py`.

The pinned expected values were generated by the **same**
`_little_mcar_test` routine that `analyze_missingness` uses internally —
they are snapshots of the current vendored implementation rather than
independently-derived ground truth.

The regression catches:

- accidental parameterization changes (e.g. ddof drift on the pooled covariance),
- per-pattern d² accumulator bugs,
- silent scipy `chi2`/`linalg` version drift.

## Dataset shapes

| File | n | numeric cols | Planting | Expected verdict |
|---|---|---|---|---|
| `ozone.csv` | 200 | 4 (ozone, solar_r, wind, temp) | independent ~10% MCAR | fail to reject (p ≈ 0.40) |
| `iris_mar.csv` | 150 | 4 (sepal_length, sepal_width, petal_length, petal_width) | sepal_width MAR ∝ petal_length | reject (p ≪ 0.01) |
| `nhanes.csv` | 200 | 4 (age, bmi, sbp, glucose) | bmi MAR ∝ age, glucose MAR ∝ sbp | reject (p ≪ 0.01) |

## TODO — swap in R's `naniar::mcar_test` reference

The proposal calls for ±1e-3 agreement with R's `naniar::mcar_test`. R
is not installed in this environment, so we pin to our own
implementation's output to get *some* regression coverage now and defer
the R cross-validation to a follow-up.

To upgrade: install R + the `naniar` package, run
`naniar::mcar_test(read.csv("<file>.csv"))` for each fixture, replace
the statistic/df/p_value values in `expected.json` with R's output, and
the existing test will tighten automatically.

## Regenerating

```bash
uv run python fixtures/_mcar_reference/generate.py
```

The generator uses fixed seeds (ozone=11, iris_mar=0, nhanes=7) so the
CSVs are stable across runs.
